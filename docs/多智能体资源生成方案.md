# 多智能体协同资源生成 — 详细实现方案

## 一、架构总览

### 1.1 编排者-专家模式 (Orchestrator-Specialist)

```
用户描述需求（专业、课程、短板、目标）
         │
         ▼
┌─────────────────────────────────┐
│     OrchestratorAgent           │
│  职责：理解需求 → 制定计划       │
│        → 分配子任务 → 聚合结果   │
│  模型：DeepSeek v4-pro          │
└─────────────┬───────────────────┘
              │ 分解为 5 个子任务
              ▼
┌──────────────────────────────────────────────────┐
│              专家 Agent 集群（分批并行）             │
│                                                   │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│  │ Course   │ │ MindMap  │ │ Exercise │  ← 第1批 │
│  │ Agent    │ │ Agent    │ │ Agent    │   (3并发) │
│  │ 课程讲解  │ │ 思维导图  │ │ 练习题   │          │
│  └────┬─────┘ └────┬─────┘ └────┬─────┘          │
│       │             │             │                │
│  ┌────┴─────┐ ┌────┴──────────┐                  │
│  │ Reading  │ │ Media Agent   │    ← 第2批        │
│  │ Agent    │ │ 教学视频/动画  │     (2并发)       │
│  │ 拓展阅读  │ │               │                  │
│  └────┬─────┘ └────┬──────────┘                  │
│       │             │                             │
└───────┼─────────────┼─────────────────────────────┘
        │             │
        ▼             ▼
┌─────────────────────────────────┐
│      ResultAggregator           │
│  职责：合并结果、质量审查、       │
│        统一格式化输出             │
└─────────────┬───────────────────┘
              │
              ▼
      SSE 流式推送到前端
```

### 1.2 为什么选这个模式

| 对比维度 | 编排者-专家 ✓ | 完全自主协作 ✗ | 固定流水线 ✗ |
|----------|:---------:|:---------:|:---------:|
| 可控性 | 高 | 低 | 高 |
| 灵活性 | 高 | 高 | 低 |
| 用户体验 | 一致 | 混乱 | 僵硬 |
| 调试难度 | 低 | 高 | 低 |
| 适合资源生产 | ✓ | ✗ | △ |

结论：资源生产场景下，各资源类型独立性强、天然并行，编排者模式是最佳拟合。

## 二、5种资源类型与Agent设计

### 2.1 CourseAgent — 课程讲解文档

| 属性 | 值 |
|------|-----|
| 输入 | 课程名、知识点列表、学生水平 |
| 输出 | 结构化Markdown文档（含目录、小节、关键概念、图示） |
| 模型 | DeepSeek v4-pro |
| 产物格式 | Markdown → 前端渲染为富文本 |
| System Prompt 要点 | "你是资深课程设计师。按教科书标准组织内容，包含学习目标、核心概念、例题、小节总结。" |

### 2.2 MindMapAgent — 知识点思维导图

| 属性 | 值 |
|------|-----|
| 输入 | 主题知识点 + 子知识点列表 |
| 输出 | Mermaid mindmap 语法 |
| 模型 | DeepSeek v4-pro |
| 产物格式 | Mermaid 代码块 → 前端用 mermaid.js 实时渲染为 SVG |
| System Prompt 要点 | "你是知识结构专家。将知识点组织为层级化思维导图，输出标准Mermaid mindmap语法。" |

**为什么用 Mermaid 而不是图片？**
- 纯文本 → 可流式展示、可编辑
- 前端渲染 → 无需额外图片生成API调用
- 支持导出 PNG/SVG

### 2.3 ExerciseAgent — 练习题

| 属性 | 值 |
|------|-----|
| 输入 | 知识点、难度要求、题型偏好 |
| 输出 | JSON结构化题库 |
| 模型 | DeepSeek v4-pro (json_mode) |
| 产物格式 | JSON → 前端渲染为交互式答题卡 |
| System Prompt 要点 | "你是命题专家。生成题目，严格按JSON格式输出。包含题干、选项、答案、解析、难度、知识点标签。" |

JSON Schema:
```json
{
  "exercises": [
    {
      "id": "ex001",
      "type": "choice|short_answer|case_analysis|code",
      "difficulty": "easy|medium|hard",
      "tags": ["知识点1", "知识点2"],
      "question": "题目内容",
      "options": ["A. ...", "B. ...", "C. ...", "D. ..."],
      "answer": "正确答案",
      "explanation": "详细解析"
    }
  ]
}
```

### 2.4 ReadingAgent — 拓展阅读材料

| 属性 | 值 |
|------|-----|
| 输入 | 主题领域、当前水平、兴趣方向 |
| 输出 | 结构化阅读推荐列表 |
| 模型 | DeepSeek v4-pro |
| 产物格式 | Markdown → 前端渲染为阅读卡片 |
| System Prompt 要点 | "你是学术导师。推荐3-5篇拓展资料，包含：论文/书籍/博文标题、作者/来源、核心观点、为什么适合当前学生、阅读建议顺序。" |

### 2.5 MediaAgent — 教学视频/动画

| 属性 | 值 |
|------|-----|
| 输入 | 知识点、视频风格（讲解/动画/实操） |
| 输出 | 视频脚本 + Seedance生成的教学视频 |
| 模型 | DeepSeek (写脚本) + Seedance doubao-seed-2-0-lite (视频生成) |
| 产物格式 | 视频文件 (OSS存储) + 脚本文本 |
| System Prompt 要点 | "你是教学视频导演。写出3-5分钟的短视频分镜脚本，语言生动、画面感强。输出分镜脚本和Seedance视频生成Prompt。" |

**注意**：MediaAgent 耗时最长（视频生成通常30s-2min），且 Seedance 已有 API Key 但未使用过。建议：
- MVP 阶段先生成**视频脚本**展示给用户
- 视频生成作为"一键生成视频"按钮，异步执行
- 完成后通过轮询或SSE通知用户

### 3.2 resources_agent.py 核心设计

```
class OrchestratorAgent:
    """编排者：理解需求 → 制定计划 → 分配任务"""

    def understand_needs(user_input, conversation_history) -> NeedsSummary:
        """对话理解学生需求，提取：专业、课程、知识点、短板、目标"""

    def create_plan(needs: NeedsSummary) -> GenerationPlan:
        """根据需求制定生成计划，决定启用哪些Agent、每个Agent的参数"""

    def aggregate(results: dict[str, AgentResult]) -> ResourcePackage:
        """聚合所有Agent结果，添加总览摘要"""


class CourseAgent:
    """课程讲解文档生成"""

class MindMapAgent:
    """思维导图生成（Mermaid语法）"""

class ExerciseAgent:
    """练习题生成（JSON结构化）"""

class ReadingAgent:
    """拓展阅读推荐生成"""

class MediaAgent:
    """教学视频脚本 + Seedance视频生成"""


# 主入口函数（被API调用）
async def generate_resources(user_input, conversation_history, enabled_agents):
    """
    主流程：
    1. OrchestratorAgent.understand_needs()
    2. OrchestratorAgent.create_plan()
    3. 分批并行执行专家Agent（ThreadPoolExecutor）
       - 第1批: Course + MindMap + Exercise (3并发)
       - 第2批: Reading + Media (2并发)
    4. OrchestratorAgent.aggregate()
    5. 通过SSE逐结果推送前端
    """
```

### 3.3 API路由设计

前缀: `/api/v1/resources`

| 方法 | 路径 | 功能 | 说明 |
|------|------|------|------|
| POST | `/start` | 开始资源生成对话 | Orchestrator理解需求，返回确认+计划 |
| POST | `/generate` | 启动多Agent生成 | 返回 resource_package_id |
| GET | `/generate/stream` | SSE流式接收生成结果 | 复用已有SSE模式 |
| GET | `/packages` | 列出用户所有资源包 | 历史记录 |
| GET | `/packages/{id}` | 获取指定资源包完整内容 | 含所有5种资源 |
| POST | `/packages/{id}/media` | 异步生成教学视频 | MediaAgent单独触发 |

### 3.4 SSE事件协议

```
event: plan
data: {"agents": ["course","mindmap","exercise","reading","media"],
       "summary": "将为你生成机器学习入门5件套"}

event: agent_start
data: {"agent": "course", "label": "课程讲解文档", "total": 5, "current": 1}

event: agent_chunk
data: {"agent": "course", "content": "## 第一章 监督学习基础\n\n..."}

event: agent_done
data: {"agent": "course", "result": {"title": "...", "content": "...", "metadata": {...}}}

event: agent_error
data: {"agent": "exercise", "error": "生成超时，请重试"}

event: all_done
data: {"package_id": 42, "summary": "全部5种资源生成完毕"}
```

### 3.5 数据库扩展

新增表 `resource_packages`:

```sql
CREATE TABLE resource_packages (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id INT NOT NULL,
    session_id INT,
    needs_summary JSON,          -- 学生需求摘要
    generation_plan JSON,        -- 生成计划
    resources JSON,              -- 所有资源内容
    status ENUM('draft','generating','done','failed') DEFAULT 'draft',
    enabled_agents JSON,         -- 启用了哪些Agent
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);
```

新增表 `generated_media`（视频单独管理，因为生成耗时长）:

```sql
CREATE TABLE generated_media (
    id INT AUTO_INCREMENT PRIMARY KEY,
    package_id INT NOT NULL,
    script TEXT,                 -- 视频脚本
    seedance_prompt TEXT,        -- Seedance生成Prompt
    video_url VARCHAR(500),      -- OSS视频URL
    status ENUM('pending','script_done','generating','done','failed') DEFAULT 'pending',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (package_id) REFERENCES resource_packages(id)
);
```

## 五、用户交互流程

### 完整流程

```
1. 用户进入 /generate 页面
   ↓
2. AI 开场："同学你好！我是学习资源生成助手。请告诉我：
   - 你的专业和年级
   - 最近在学什么课程
   - 哪些知识点比较薄弱
   - 你想达到什么学习目标"
   ↓
3. 用户回复（自由文本）
   ↓
4. OrchestratorAgent 理解需求 → 返回确认：
   "我理解了，你是大二计算机专业，正在学《机器学习》，在
    梯度下降和过拟合方面需要加强，目标是通过期末考试。
   
   我计划为你生成以下资源：
   → 机器学习核心课程讲解文档
   → 知识点思维导图
   → 梯度下降+过拟合专项练习题
   → 拓展阅读推荐
   → 教学讲解视频
   
   是否开始生成？你也可以告诉我想调整的地方。"
   ↓
5. 用户确认/调整 → 点击"开始生成"或回复"好的"
   ↓
6. 后端启动多Agent并行生成 → SSE推送进度
   前端显示 ResourceGenerating 动画
   ↓
7. 首个资源完成（通常是CourseAgent最快）→ Tab自动激活
   用户可以边看已完成的资源边等待其余生成
   ↓
8. 全部完成 → AI总结："5种资源已生成完毕！你可以在左侧
   边栏找到历史资料包。有需要修改的随时告诉我。"
```

## 六、模型调用策略

### 6.1 并发控制

由于所有LLM走同一代理 `127.0.0.1:7897`，需控制并发：

```
第1批（3并发）：CourseAgent + MindMapAgent + ExerciseAgent
    ↓ 等待第1批全部完成
第2批（2并发）：ReadingAgent + MediaAgent
```

- 如果某个Agent超时（>60s），自动重试1次
- 如果重试仍失败，标记为 `agent_error`，其他Agent不受影响
- 用户可单独点击"重新生成"失败的资源

### 6.2 各Agent模型分配

| Agent | 模型 | 理由 |
|-------|------|------|
| Orchestrator | DeepSeek v4-pro | 需要强推理能力来理解需求、制定计划 |
| CourseAgent | DeepSeek v4-pro | 长文生成，DeepSeek文本质量好 |
| MindMapAgent | DeepSeek v4-pro | 结构化输出，DeepSeek遵循指令好 |
| ExerciseAgent | DeepSeek v4-pro | JSON结构化，需要精确遵循Schema |
| ReadingAgent | DeepSeek v4-pro | 文本推荐，跟CourseAgent类似 |
| MediaAgent (脚本) | DeepSeek v4-pro | 创意写作 |
| MediaAgent (视频) | Seedance | 视频生成专用模型 |

所有Agent共用DeepSeek，通过System Prompt实现角色分化。

## 九、风险评估

| 风险 | 概率 | 影响 | 应对 |
|------|------|------|------|
| 5个Agent并发→API限流 | 中 | 生成变慢 | 分2批执行，每批间隔3s |
| Seedance视频生成不稳定 | 高 | MediaAgent失败 | MVP先出脚本，视频异步补生成 |
